# Agent = LLM + 记忆 + 规划 + 工具使用 + 循环执行
## 规划
思维链（CoT）
思维链(CoT，2022)指示模型“逐步思考”，以利用更多测试时计算将复杂任务分解为更小、更简单的步骤。
CoT将大任务转化为多个可管理的任务，并阐明模型思维过程的解释。

思维树(ToT，2023)通过在每一步探索多种推理可能性来扩展思维树。它首先将问题分解为多个思考步骤，并在每一步生成多个思考，从而创建一个树形结构。搜索过程可以是广度优先搜索(BFS)或深度优先搜索(DFS)，每个状态由分类器(通过提示)或多数表决进行评估。

大模型+规划工具(LLM+P，2023)用规划域定义语言(PDDL)作为中间接口来描述规划问题。在此过程中，LLM(1)将问题转换为"问题PDDL”，然后(2)请求经典规划器基于现有的“域PDDL”生成PDDL规划，最后3)将PDDL规划转换回自然语言。本质上，规划步骤外包给外部工具，假设有特定于领域的PDDL和合适的规划器，这在某些机器人设置中很常见，但在许多其他领域并不常见。

推理+执行(ReAct，2023)将推理和行动结合起来应用到LLMs上。其中，推理产生的traces帮助模型归纳、跟踪、更新行动计划，并辅助模型进行异常处理;行动则通过与知识库、维基百科APl、环境等外部信息源交互来收集信息。推理与行动交错进行，不仅改善了模型的幻觉(hallucination)和错误传播(error propagation)问题，同时使得LLMs解决问题的过程本身(trajectories)更加接近人类表现，即更具有可解释性，从而生成的结果也更加可信。

自我反思(Reflexion，2023)收到ReAct 的这种基于上下学习低成本快速调整LLM的思路启发，使用语言反馈信号来帮助agent从先前的失败经验中学习。具体地，Reflexion将传统梯度更新中的参数信号转变为添加在大模型上下文中的语言总结，使得agent在下一个episode中能参考上次执行失败的失败经验，从而提高agent的执行效果。这个过程和人类反思(reflexion)过程十分相似。

无需观察的推理(ReWO0，2023)与ReAct不同，ReWO0省去了观察步骤，而是提前规划。这种智能体推理设计模式包含三个模块:规划器(Planner)、执行器(Worker)和求解器(Solver).规划器模块将任务分解为子任务，并将每个子任务分配给一个工作模块。工作模块将用于证实每个子任务的工具与证据和事实相结合。最后，求解器模块综合所有子任务及其相应的证据，得出结论。==（速度的提升）==


## 记忆

感觉记忆:作为原始输入的学习嵌入表示，包括文本、图像或其他模态;
短期记忆:作为情境学习。由于受到Transformer有限上下文窗口长度的限制，它既短又有限。
长期记忆:作为智能体在查询时可以关注的外部向量存储，可通过快速检索访问。

## 工具
![[Pasted image 20260610232128.png]]
自己写函数，大模型需要时自己调用


LLM 创作工具 (LLMs as Tool Makers, 2023)为避免对外部工具可用性(availability)的依赖，本文提出一种让LLMs自己制造工具，并使用或复用该工具的闭环框架(closed-loop framework)。其中，工具制造者和使用者可以是同一个LLM，也可以是不同LLM。这里，工具特指基于Python实现的工具函数。

LLM即工具(LLMas Tool)是把一个LLM看作是工具通过特定的方式(如MCP、A2A)来使用另一个LLM或Aaent.
